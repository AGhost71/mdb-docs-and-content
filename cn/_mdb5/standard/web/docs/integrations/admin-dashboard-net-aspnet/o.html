先决条件

在启动项目之前，请确保安装以下实用程序：

  • .NET Core (3.1.x)
  • Entity Framework Core .NET Command-line Tools (3.1.x)
  • Code editor. We recommend VSCode

安装

下载我们的模板后，您需要安装所需的软件包。

导航到您的项目并在终端运行中：

点网还原

数据库配置

1。。创建数据库

为了使我们的应用正常运行，我们需要创建一个MySQL数据库（在本示例中，我们将其称为 仪表板，但您可以稍后进行更改）。

在本说明中，我们将不介绍有关MySQL数据库以及如何设置它们的详细信息- 您可以在此处了解更多信息：

  • MySQL docs
  • MariaDB

2。连接字符串

要连接数据库，我们需要在appsettings.json中设置连接字符串。 文件：

"ConnectionStrings": { "AppConnection": "server=localhost;user=test;password=test;database=dashboard" }

用您的root用户（不推荐！）替换用户和密码值或 为此数据库创建的凭据（推荐）。

注意：默认端口设置为3306-如果您希望更改它，则可以 需要在您的连接字符串中指定端口的值。

Learn more about connection strings here.

3。。应用迁移

现在，当一切设置完成后，我们可以更新数据库以匹配我们的初始迁移。我们 从现在开始将使用Entity Framework Core工具（了解更多 此处）。

在您的终端运行中：

dotnet-ef database update

刷新数据库后，您应该会看到4个新表（角色，用户，路况数据和 迁移历史）。

每次您希望更新数据库（更改/添加模型）时，都需要创建一个新的数据库。 移民：

dotnet-ef迁移添加migrationName dotnet-ef数据库更新

运行应用程序

现在是时候运行我们的应用程序了。在控制台中，运行以下命令：

dotnet run

注意：应用程序将在两个端口上启动-5000（http）和5001（https）。

用法

我们的模板为您的应用程序提供了现成的几种功能：

1。。登录/注销

该应用程序仅可供登录用户访问-该应用程序使用Cookie来 授权用户并检查凭据（管理员具有其他特权）。

注销将删除cookie并将用户重定向到登录页面。

2。注册

您可以使用注册表单添加新用户。虽然无法注册 具有一个电子邮件地址的多个用户，则不提供电子邮件确认功能。

3。用户个人资料

每个用户都可以使用用户上的表格更新其用户个人资料（头像，用户名，密码） 个人资料页。

4。。用户管理（仅管理员）

具有管理员权限的用户可以更新有关其他用户的信息。在我们的基本应用中 我们可以更改用户的凭据。

默认值

应用初始迁移时，应用程序将播种默认数据。

您可以在 Data / DataContext.cs 中管理此数据

1。用户

管理员：

{ name: "Admin", email: "admin@mdbootstrap.com", password: "admin", roleId: 1 }

User:

{ name: "User", email: "user@mdbootstrap.com", password: "user", roleId: 2 }

2. Roles

  • Admin (id: 1)
  • User (id: 2)

3。流量数据

常规仪表板的五个随机生成的条目。

后端功能

1。 ORM和模型

该应用程序使用实体框架核心（MySQL）。您可以在中管理DbContext 数据/DataContext.cs

开箱即用的应用程序中可用的模型：

  • Users
  • Role
  • TrafficData

2。数据传输对象

创建数据传输对象对于前端-后端通信很重要，因为有些 客户端上不能访问该字段（例如密码）。

要将数据轻松地从模型映射到DTO（以及其他方法），请定义所需的配置文件 （例如：Profiles / UserProfile.cs）。

3. Controllers & routes

Controllers:

  • HomeController
  • AuthController
  • UserController
  • PerformanceController

Home Controller

Rendering views / redirecting unauthorized users.

Routes:

  • /Home/UserManagement
  • /Home/Index
  • /Home/UserProfile
  • /Home/Dashboard

Auth Controller

Authorization management

Routes & actions:

  • /auth/login (POST, request body: loginDTO)
  • /auth/register (POST, request body: createUserDTO)
  • Logout()

Performance Controller

The controller provides data for the Dashboard View.

Routes:

  • /performance/traffic (GET)
  • /performance/traffic (POST) - adds randomly generated entry to the traffic data table

前端功能

1. Views

Views for your application:

  • Index - Login / Register form
  • Dashboard - General Dashboard
  • UserManagement - Admin panel
  • UserProfile - User info & password management

2. Layouts

There are two layouts included in the project:

  • Layout (main layout for authorized users)
  • LoginLayout (layout for unlogged user, without navigation)

3. wwwroot folder

  • /assets - additional files, for example default avatar
  • /mdb - MDB 5 files
  • /js - js files for views
  • layout.css - custom css for layouts
