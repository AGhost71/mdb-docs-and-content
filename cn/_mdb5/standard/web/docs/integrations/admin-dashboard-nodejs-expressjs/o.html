先决条件

在启动项目之前，请确保安装以下实用程序：

  • Node LTS (12.x.x recommended)
  • 代码编辑器。我们推荐 VSCode

安装

要安装项目，请按照下列步骤操作：

  • 下载并解压缩 高级套餐
  • 在 / integrations / admin-dashboard-node-express 目录中运行终端， 放在包装内
  • 在终端中输入 npm install

数据库安装

要操作数据库，我们将需要Web服务器实用程序。我们推荐 XAMPP 。请记住，只有 Apache和MySQL模块是必需的。这是指南：

  • Install XAMPP and run as an administrator
  • Make sure that Appache and MySQL modules are running
  • MySQL should work on port 3306 (it's the default option)
  • Move to the phpmyadmin page (Admin button in the MySQL section) and create a new database called mdb_admin_dashboard_node
  • If you want to change MySQL db config, please update the .env file in the admin dashboard root directory. It contains the configuration of your database and app port. The defaults are:
    • DB_HOST=localhost
    • DB_NAME=mdb_admin_dashboard_node
    • DB_USER=root
    • DB_PASS=
    • PORT=3000

运行应用程序

要运行该项目，请按照下列步骤操作：

  • 在 / integrations / admin-dashboard-node-express 目录中运行终端
  • 在终端中输入 npm start
  • 应用程序应创建一些默认的数据库记录并返回消息： 服务器在端口3000上运行。
  • 您现在可以在浏览器中打开 http：// localhost：3000 / 页面

用法

Main view (not logged in user):

  • Panel with the login form for already registered users. By default there are two users:
    • email: admin@mdbootstrap.com, password: mdbootstrap, role: Admin
    • email: user@mdbootstrap.com, password: mdbootstrap, role: User
  • Panel with the registration form with client and server side validation.

Dashboard view (logged in user):

  • Sample content with some SEO statistics. The last chart and table are filled with the data from the MySQL table called traffic_data
  • Sidenav with links to My profile and User management (only with Admin role) pages
  • Navbar with user avatar and logout button

My profile page:

  • Section with user data (name, avatar) with an edit option
  • Section with user password change

User management page (only with Admin role):

  • Section with all users data
  • Ability to remove any user
  • Ability to change roles

默认值

默认情况下，应用程序创建三个MySQL表：

users

  • id
  • name
  • email
  • password
  • avatar
  • role_id
  • created_at
  • updated_at

roles

  • id
  • name
  • created_at
  • updated_at

traffic_data

  • id
  • impressions
  • clicks
  • position
  • created_at
  • updated_at

上面的每个表默认情况下都填充有示例内容。有两个用户 （admin @ mdbootstrap.com，user @ mdbootstrap.com），两个角色（管理员，用户）和五个随机角色 SEO信息中心的流量数据记录。

后端功能

ORM (Object-Relational-Mapper):

在此应用程序中，我们使用Sequelize。Sequelize是Postgres的基于诺言的Node.js ORM， MySQL，MariaDB，SQLite和Microsoft SQL Server。它具有可靠的交易支持， 关系，急切和延迟加载，读取复制等。可以找到文献 此处

Models:

模型是Sequelize的本质。模型是代表您表格中的表格的抽象 数据库。在Sequelize中，它是扩展Model的类。该模型告诉Sequelize几个 有关它表示的实体的信息，例如数据库中表的名称以及 它具有的列（及其数据类型。Sequelize中的模型有一个名称。此名称没有 与其在数据库中表示的表的名称相同。通常，模型有单数 名称（例如User），而表具有复数名称（例如Users），尽管这是 完全可配置的。

在我们的项目中，存在三个模型类：用户，角色，TrafficData 。

路线和控制器:

路由是Express代码的一部分，它与HTTP动词（GET，POST，PUT，DELETE， 等等），URL路径/模式以及调用该模式的函数。有 创建路线的几种方法。在我们的应用中，我们尽可能使用express.Router中间件 我们将站点特定部分的路由处理程序分组在一起，并使用 通用的路由前缀。控制器代表用于添加模型的方法 功能，例如创建，更新或删除实例。

中间件:

中间件功能是可以访问请求对象（req），响应的功能 对象（res），以及应用程序的请求-响应周期中的下一个中间件功能。 下一个中间件功能通常由名为next的变量表示。中间件 函数可以执行以下任务：

  • Execute any code
  • Make changes to the request and the response objects
  • End the request-response cycle
  • Call the next middleware function in the stack

如果当前的中间件功能没有结束请求-响应周期，则必须调用 next（）将控制权传递给下一个中间件函数。否则，请求将被保留 挂。

前端功能

MDB UI套件:

该项目包括我们的ui工具包，您可以使用它快速构建基本视图。差不多所有 应用程序布局是使用 MDB管理仪表板模板.

EJS （嵌入式JavaScript模板）：

EJS是一种简单的模板语言，可让您使用纯JavaScript生成HTML标记。 对如何组织事物没有虔诚。无需重新设计迭代和控制流。 这只是普通的JavaScript。我们使用EJS来显示所有前端视图和布局。欲了解更多 信息请访问 此文档页面

视图和布局:

所有前端内容都放置在 views 目录中。有两个主要页面 （索引，仪表板）和另外五种布局（页脚，导航栏，sidenav，seo仪表板， 用户个人资料，用户管理）。多亏了EJS，我们可以有条件地显示一些内容（例如 取决于用户的角色）。
