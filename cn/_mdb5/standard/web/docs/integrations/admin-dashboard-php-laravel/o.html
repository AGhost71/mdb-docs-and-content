先决条件

  • PHP和MySQL的本地Apache环境
  • C作曲家： https://getcomposer.org/
  • Laravel版本8.x: https://laravel.com/docs/8.x#installing-laravel

安装

下载完模板后，导航至项目文件夹并运行以下命令 在终端：

  1. composer dump-autoload
  2. composer install
  3. npm install
  4. cp .env.example .env

数据库配置

1。。创建数据库

为了使应用程序正常运行，您需要创建一个新的MySQL数据库。为了 为了本指南的目的，我们将其命名为 mdb_admin_dashboard_laravel ，但是您可以 在此处使用您的自定义数据库名称。

在本说明中，我们将不介绍有关MySQL数据库以及如何设置它们的详细信息- 您可以在此处了解更多信息：

  • MySQL docs
  • MariaDB

2。。配置连接

要连接到数据库，我们需要在数据库中配置连接参数 .env 文件：

DB_CONNECTION=mysql DB_HOST=127.0.0.1 DB_PORT=3306 DB_DATABASE=mdb_admin_dashboard_laravel DB_USERNAME=root DB_PASSWORD=

用您的凭据替换用户名和密码值。如果您使用 自定义名称的第一步，将其粘贴到 DB_DATABASE 字段中。除此以外， 保留默认的 mdb_admin_dashboard_laravel 名称。

3。迁移

现在，当一切设置完成后，我们可以运行自动迁移来创建所需的表 和数据库中的列。

在您的终端运行中：

php artisan migrate

刷新数据库后，应该会看到新的表和列。

4。

Laravel提供了一种简单的方法，可以使用测试数据来为您的数据库添加种子。

在您的终端运行中：

php artisan db:seed

刷新数据库后，应该会看到测试数据。

运行应用程序

现在是时候运行我们的应用程序了。在控制台中，运行以下命令：

php artisan serve

注意：该应用程序将在端口8000上启动。

用法

我们的模板为您的应用程序提供了现成的几种功能：

1。。登录/注销

该应用程序的完整版本仅适用于登录用户（也为admin用户） 有权访问用户管理面板）。该应用程序使用基于Cookie的 身份验证。

注销后，Cookie将被删除，并且用户将被重定向到登录/注册 页。

2。注册

您可以通过填写注册表格来添加新用户。使用管理员创建用户 特权，您需要手动将具有特定 role_id 的用户添加到 数据库中的 users 表。

3。用户个人资料

每个用户都可以通过在表单中提交表单来更新其个人资料（头像，用户名，密码） 用户个人资料视图。

4。。用户管理（仅管理员）

具有管理员权限的用户可以更新有关其他用户的信息。

默认值

应用初始迁移时，应用程序将播种默认数据。您可以更改 database / seeders / DatabaseSeeder.php 文件中的默认设置。

1。用户

默认情况下，播种机将创建五个具有随机数据的普通用户和一个管理员用户 具有以下凭证：

{ name: "Admin", email: "admin@mdbootstrap.com", password: "mdbootstrap", roleId: 1 }

2。角色

  • Admin (id: 1)
  • User (id: 2)

3。流量数据

仪表板视图中的图表和表中使用了五个随机生成的条目。

后端功能

1。模型

开箱即用的应用程序中可用的模型：

  • Users
  • Role
  • TrafficData

2。控制器

控制器：

  • LoginController - user login management
  • RegisterController - user registration management
  • DashboardController - rendering dashboard view
  • ProfileController - rendering user profile view, profile data updates management
  • UsersController - rendering admin panel view, user data updates management

前端功能

1. Views

Views for your application:

  • Index - Login and register forms
  • Dashboard - General Dashboard
  • UserProfile - User data and password management
  • UserManagement - Admin panel

2。公共文件夹

  • /mdb - MDB 5 files
  • /js - js files for views
  • /css - css files for views
