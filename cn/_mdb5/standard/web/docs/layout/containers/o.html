容器是Bootstrap的基本构建块，包含，填充和对齐您的 给定设备或视口中的内容。

这个怎么运作

容器是Bootstrap中最基本的布局元素， 使用我们的默认网格系统时需要。容器用来容纳 垫，并且（有时）将内容放在其中。尽管可以嵌套容器 ， 大多数布局不需要嵌套容器。

Bootstrap带有三个不同的容器：

  • .container ，它在每个响应断点处设置一个 max-width
  • .container-fluid ，在所有断点处的宽度为 width：100％
  • .container- {breakpoint} ，直到指定的宽度为 width：100％ 断点

默认容器

我们的默认 .container 类是一个响应式，固定宽度的容器，这意味着 最大宽度在每个断点处更改。

响应式容器

响应式容器使您可以指定100％宽的类，直到指定了 达到断点，此后，我们为每个较高的断点应用 max-width s 断点。例如， .container-sm 的宽度为100％，直到 达到了 sm 断点，它将以 md 进行扩展， lg ， xl 和 xxl 。

Sass

如上所示，Bootstrap会生成一系列预定义的容器类，以帮助您构建 您想要的布局。您可以通过修改 Sass映射（可在 _variables.scss 中找到）为它们提供动力：

$container-max-widths: ( sm: 540px, md: 720px, lg: 960px, xl: 1140px, xxl: 1320px );

除了自定义Sass，您还可以使用我们的Sass创建自己的容器 mixin。

// Source mixin @mixin make-container($padding-x: $container-padding-x) { width: 100%; padding-right: $padding-x; padding-left: $padding-x; margin-right: auto; margin-left: auto; } // Usage .custom-container { @include make-container(); }
