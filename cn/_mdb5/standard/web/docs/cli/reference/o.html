这些是每个MDB CLI命令的描述。您还可以在中查看基本帮助 您的终端与 mdb help.

mdb help

显示帮助提示.

例子 $ mdb help

mdb update

更新 mdb-cli 打包到最新版本。它会问你什么包 经理您首先使用。

例子 $ mdb update

mdb version|-v|--version

检查当前安装的版本 mdb-cli 包。它会问你 您首先使用什么程序包管理器。

例子 $ mdb version $ mdb -v $ mdb --version

mdb register

创建您的MDB帐户。它将要求您提供姓名，用户名，电子邮件和密码。 您将自动登录。

例子 $ mdb register

mdb login [OPTIONS]

使用您的MDB帐户登录到CLI。它将要求您提供用户名和 密码。如果您使用--method=[provider] 标记，它将打开浏览器窗口， 要求身份验证成功后，粘贴显示在屏幕上的代码。

选项 --method 使用社交媒体登录。可能的值：google，facebook，twitter 例子 $ mdb login $ mdb login --method=google $ mdb login --method=facebook $ mdb login --method=twitter

mdb logout

从CLI注销。

例子 $ mdb logout

mdb list

列出您帐户中可用的软件包。如果可用，是 则显示 No （否则，将显示产品页面的链接）。
这些是可用于初始化项目的软件包。

例子 $ mdb list

mdb init [OPTIONS]

使用您帐户中可用的软件包之一来初始化启动项目。列表 使用 mdb list 命令可以看到其中的软件包。您将无法 使用不可用的程序包初始化项目。

选项 -n, --name set the name of your project right after initializing it -b, --blank initialize blank project 例子 $ mdb init $ mdb init -n my-first-project $ mdb init --blank

mdb set-name [name]

设置您的项目的名称。它只会在您本地的本地更改名称 package.json 文件。更改将在远程服务器上生效 运行 mdb publish 命令。

注意
运行此命令后，第一个 mdb publish 将导致您的新副本 以新名称发布的项目。具有旧名称的旧项目仍会 存在。为了重命名项目，您需要运行 mdb重命名命令。

用法 $ mdb set-name [name] 例子 $ mdb set-name $ mdb set-name new-project-name

mdb create

在以下位置创建Git存储库 我们的GitLab服务器 当前的项目。它允许您使用MDB Go管道在我们的远程服务器上部署项目 服务器并跟踪项目构建。

用法 $ mdb create

mdb publish [OPTIONS]

将您当前的项目上传到我们的远程服务器，并通过类似的链接进行访问： https://mdbgo.dev/<username>/<project-name>/.
使用此命令可以上传新项目或使用最新版本更新现有项目 变化。

您可以通过两种方式发布项目。

首先推荐使用MDB Go管道。这意味着您的项目将使用 GitLab存储库和Jenkins作业将部署在我们的FTP服务器上。简而言之，使用这个 选项意味着您的项目将配置CI / CD管道，并具有所有优势。它 默认情况下是这样的。

第二种方法是直接将文件直接上传到FTP服务器。这样一来你就失去了一切 为您的项目配置CI / CD的优势。为了那样走，你 需要添加-ftp 参数。

选项 --ftp 不要使用MDB Go管道。超越自动构建选项，将直接上传到远程服务器 --test, -t 发布之前，运行在“ package.json”文件中定义的“测试”脚本。如果脚本返回非零退出代码，则该项目将不会发布 例子 $ mdb publish $ mdb publish -t $ mdb publish --ftp $ mdb publish --ftp -t

mdb unpublish [name]

从远程服务器上删除您的项目。

注意
如果您使用的是MDB Go管道，则您的项目仍将作为GitLab存储库存在。 詹金斯的工作也将保持不变。但是，如果您不使用我们的CI / CD设置，则您 运行此命令后，将仅具有项目的本地副本。

USAGE $ mdb unpublish [name] EXAMPLES $ mdb unpublish $ mdb unpublish my-project-name

mdb projects

显示所有已发布项目的列表。

EXAMPLES $ mdb projects

mdb get [name]

将您的项目克隆到本地计算机上。

注意
此命令仅适用于使用我们的GitLab存储库的项目。您 无法下载未连接GitLab存储库的项目。 要创建GitLab存储库并将其连接到您的项目，请使用 mdb create 命令。

用法 $ mdb get [name] 例子 $ mdb get $ mdb get my-project-name

mdb rename [name]

在本地和远程重命名当前项目。

注意
此命令将更改 package.json 文件中和 远程服务器。本地项目目录名称将保持不变。

用法 $ mdb rename [name] 例子 $ mdb rename $ mdb rename new-project-name

mdb set-domain-name [name]

设置自定义域名，当前项目将在该自定义域名下可用。需要你 在域名提供商设置中配置DNS记录（A）。

注意
此命令将仅修改 package.json 文件。更改将生效 在运行 mdb publish 命令之后。

USAGE $ mdb set-domain-name [name] EXAMPLES $ mdb set-domain-name $ mdb set-domain-name example.com

mdb unset-domain-name

取消当前项目的域名。

注意
此命令将仅修改 package.json 文件。更改将生效 在运行 mdb publish 命令之后。

EXAMPLES $ mdb unset-domain-name

mdb orders

在MDB站点上显示您的订单列表。

EXAMPLES $ mdb orders
